# FinSys BDD Automation Framework

## 📝 Description
I worked on a Test Automation Framework for a financial management web application called FinSys.
FinSys is a comprehensive financial web application internally used for managing financial data, business entities, and company records. 
The goal was to build a Cucumber BDD (Behavior Driven Development) framework from scratch using Java, Selenium WebDriver,
and JUnit — where test cases are written in plain English so that both technical and non-technical stakeholders can
understand them.

This repository contains a robust, scalable **Test Automation Framework** built from scratch to automate the end-to-end functionality of the FinSys web application. The framework follows a **Behavior Driven Development (BDD)** approach using Cucumber and utilizes the **Page Object Model (POM)** design pattern. This ensures that UI interactions and test logic are strictly separated, promoting maximum code reusability and easier maintenance.

## 🛠️ Tech Stack & Tools Used
- **Programming Language:** Java 17
- **Core Automation Engine:** Selenium WebDriver (v4.x)
- **BDD Framework:** Cucumber (v7.x)
- **Design Pattern:** Page Object Model (POM)
- **Test Runner:** JUnit
- **Build & Dependency Management:** Maven
- **Reporting:** Extent Reports & Cucumber HTML Reports
- **Version Control:** Git & GitHub
- **CI/CD:** Jenkins
- **Frontend Challenges Handled:** Extensively handles hidden DOM elements generated by the **EasyUI** JavaScript library using specialized `JavascriptExecutor` utilities.

## 📂 Framework Architecture
The project follows an industry-standard directory structure for modularity and scalability:
- `src/main/java/`: Core utilities (Future expansions).
- `src/test/java/pages/`: Contains all Page Object classes encapsulating WebElements and action methods.
- `src/test/java/steps/`: Contains Cucumber Step Definitions mapping Gherkin scenarios to Java code.
- `src/test/java/utils/`: Contains framework utilities (`DriverManager`, `ConfigReader`, `Hooks`).
- `src/test/java/runner/`: Contains the `TestRunner` class configured with JUnit.
- `src/test/resources/features/`: Contains all `.feature` files written in Gherkin (Given/When/Then).
- `src/test/resources/config.properties`: Externalized configuration properties (browser type, timeouts, URLs).

## 🚀 Roles & Responsibilities
As a QA Automation Engineer working on this framework, my core responsibilities included:
- **Requirement Analysis:** Collaborating with business stakeholders and developers to understand requirements and converting them into easily understandable Gherkin scenarios in Feature files.
- **Framework Development:** Architecting and maintaining the automated test scripts using Java, Selenium WebDriver, and the Page Object Model (POM) from scratch.
- **Handling Complex UI Libraries:** Writing robust custom utility methods using `JavascriptExecutor` to successfully interact with complex, hidden frontend elements dynamically generated by the EasyUI library.
- **Dynamic Data Management:** Implemented dynamic data generation utilities (timestamps, random IDs) to prevent backend database conflicts (e.g., "Duplicate Entry" SQL errors) during continuous test execution.
- **Test Execution & Reporting:** Generating detailed HTML test execution reports using Extent Reports and sharing actionable metrics with the development team.
- **Pipeline Monitoring:** Executing tests locally using Maven commands (`mvn clean test`) and actively tracking CI/CD pipeline stability by analyzing Jenkins execution logs.
- **Defect Tracking:** Identifying bugs during automation script creation/execution and reporting them with clear reproducible steps in bug-tracking tools like Jira.

## ▶️ Execution Instructions
**1. Local Execution via IDE:**
- Open `src/test/java/com/finsys/runner/TestRunner.java`.
- Right-click the file and select `Run As -> JUnit Test`.

**2. Maven Execution via Terminal:**
```bash
# To clean previous builds and execute the test suite
mvn clean test
```
